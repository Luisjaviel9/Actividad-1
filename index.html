Cuestionario Sobre el contenido de la Unidad I.


1. ¿Qué es un Sistema Operativo?

Un Sistema Operativo (SO) es un conjunto de programas que gestionan y coordinan el hardware y el software de una computadora. Proporciona una interfaz entre el usuario y los recursos físicos de la máquina, permitiendo la ejecución de aplicaciones y la gestión de tareas.

2. Componentes de un Sistema Operativo

Los componentes principales de un SO
Núcleo (Kernel): Maneja los recursos del sistema y la comunicación entre hardware y software.
Gestor de procesos: Administra la ejecución de programas.
Gestor de memoria: Controla la asignación y liberación de la memoria.
Sistema de archivos: Organiza y almacena los datos.
Interfaz de usuario: Puede ser gráfica (GUI) o de línea de comandos (CLI).

3. Tareas del Sistema Operativo

• Gestión de procesos.
• Gestión de memoria.
• Gestión de archivos.
• Control de dispositivos.
• Administración de usuarios y permisos.
• Manejo de errores y excepciones.

4. Funciones de un Sistema Operativo

• Facilitar la comunicación entre el usuario y la computadora.
• Administrar recursos de hardware y software.
• Proporcionar una plataforma para ejecutar aplicaciones.
• Asegurar la seguridad y el acceso controlado a los datos.

5. Características de un Sistema Operativo

• Multitarea: Capacidad de ejecutar múltiples tareas simultáneamente.
• Multiusuario: Permite a varios usuarios utilizar la computadora al mismo tiempo.
• Portabilidad: Puede ser utilizado en diferentes tipos de hardware.
• Seguridad: Protección de datos y control de acceso.

6. Tipos de Sistema Operativo

• Monousuario y monoproceso: SO que solo puede manejar un usuario y una tarea a la vez.
• Monousuario y multitarea: SO que permite un usuario con múltiples tareas.
• Multiusuario y multitarea: SO que permite múltiples usuarios y múltiples tareas.
• Distribuidos: SO que coordina varios sistemas computacionales conectados.
• Tiempo real: SO que responde a eventos en tiempo real.

9. Sistema Operativo según el Ordenador y el Tipo de Aplicaciones

El tipo de SO varía según el hardware (PC, servidores, dispositivos móviles) y el tipo de aplicaciones que se ejecutarán (juegos, desarrollo, servidores web, etc.).

10. Compatibilidad de los Sistema Operativo

La compatibilidad de un SO depende del hardware que soporta y de las aplicaciones que puede ejecutar. Un SO debe ser compatible con los controladores y el software necesario para funcionar correctamente.

11. ¿Cuál es la función básica de un Sistema Operativo?

La función básica de un SO es gestionar los recursos del hardware y proporcionar una interfaz para que los usuarios y las aplicaciones puedan interactuar con el sistema de manera eficiente y segura.

12. Mencione tres Sistemas Operativos más utilizados

• MacOS

13. Mencione algunas tareas que realizan los Sistemas Operativos

• Gestión de archivos: Crear, leer, escribir y eliminar archivos.
• Gestión de memoria: Asignar y liberar memoria para aplicaciones.
• Gestión de procesos: Crear, ejecutar y finalizar procesos.
• Control de dispositivos: Administrar la entrada y salida de dispositivos de hardware.

14. ¿Qué es un sistema operativo multitareas?

Un sistema operativo multitareas es aquel que permite ejecutar múltiples tareas o procesos al mismo tiempo, gestionando eficientemente los recursos del sistema.

15. ¿Qué es un sistema operativo multiusuario?

Un sistema operativo multiusuario permite que varios usuarios accedan y utilicen los recursos de una computadora simultáneamente, manteniendo la seguridad y la integridad de los datos de cada usuario.

16. ¿Cómo funciona un sistema operativo?

Un sistema operativo funciona como un intermediario entre el usuario y el hardware de la computadora. Gestiona las solicitudes de entrada y salida, asigna recursos, y ejecuta y supervisa programas y aplicaciones.

17. ¿Cómo se utiliza un sistema operativo en un entorno gráfico?

En un entorno gráfico, un usuario interactúa con el SO a través de interfaces gráficas (GUI), usando dispositivos de entrada como el ratón y el teclado para señalar y hacer clic en objetos que aparecen en la pantalla. Esto es manejado por el gestor de ventanas y otros componentes del SO.

18. ¿Cuándo inicia el funcionamiento del Sistema Operativo?

El SO comienza a funcionar cuando el BIOS (Basic Input/Output System) ha completado su trabajo inicial de verificación del hardware y ha cargado el SO en la memoria al encenderse o reiniciar la computadora.

19. Mencione las dos interfaces que forman parte del Sistema Operativo

• Interfaz gráfica de usuario (GUI): Permite interactuar con el SO mediante gráficos y ventanas.
• Interfaz de línea de comandos (CLI): Permite interactuar con el SO mediante comandos de texto.

20. ¿Qué es un proceso en el entorno de los sistemas operativos?

Un proceso es un programa en ejecución que ha sido cargado en la memoria y está siendo gestionado por el SO. Incluye el código del programa, los datos, y el estado del proceso.

21. ¿Qué es un sistema operativo distribuido y centralizado?

• Sistema Operativo Distribuido: Permite utilizar recursos de múltiples computadoras conectadas en red, coordinando su funcionamiento como si fueran un solo sistema.
• Sistema Operativo Centralizado: Utiliza los recursos de una sola computadora para realizar todas las tareas y servicios.

22. Mencione los sistemas de archivos utilizados por los Sistemas Operativos

• Windows: NTFS, FAT32
• Windows Vista: NTFS
• Linux: ext4, ext3, Btrfs
• MacOS: APFS, HFS+

23. Mencione otras de las características actuales de los sistemas operativos

• Soporte para múltiples procesadores. • Capacidad de virtualización. • Seguridad avanzada y encriptación. • Interfaz de usuario amigable y personalizable. • Integración con servicios en la nube

24. ¿Qué permite la existencia de muchos controladores en un sistema operativo?

Permite la rápida compatibilidad con una amplia variedad de dispositivos de hardware, asegurando que el SO pueda funcionar con diferentes componentes y periféricos sin problemas.
